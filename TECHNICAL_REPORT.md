## 1 绪论

### 1.1 研究背景
随着无人机（Unmanned Aerial Vehicle, UAV）技术的快速发展，低空空域的数字化和精细化管理成为行业刚需。传统的无人机地面站（Ground Control Station, GCS）通常依赖二维地图进行航迹规划和态势展示，难以直观反映无人机在复杂城市环境或山地地形中的三维空间关系。通过摄影测量（Photogrammetry）技术获取的高精度三维数字表面模型（Digital Surface Model, DSM）为解决这一问题提供了数据基础。

### 1.2 研究内容
本项目的核心内容是研发一套能够将外部无人机的实时遥测数据与静态的摄影测量三维模型进行虚实融合显示的系统（Demo-01）。系统需要解析高频的无人机状态数据，并在同一三维渲染管线中，将无人机实体、历史轨迹与真实世界的三维重建网格进行同帧渲染。

### 1.3 研究方法
项目采用跨平台 C++ 框架 Qt6 构建图形用户界面，利用 OpenGL 图形接口进行三维渲染。通过 Assimp 库加载复杂的摄影测量多边形网格（Polygon Mesh），并通过 UDP 网络协议实现与外部设备或模拟器的低延迟数据通信。

### 1.4 技术突破点
(1) 实现了基于 Assimp 的大规模不规则三角网（Triangulated Irregular Network, TIN）的高效加载与渲染。
(2) 设计了轻量级的 JSON 遥测协议，支持 20Hz 以上的高频位姿更新。
(3) 建立了统一的局部场景坐标系（SCENE），为未来接入 WGS84 大地坐标系奠定了基础。

### 1.5 验证与分析
通过内置的 Python 遥测模拟器，生成带有高度起伏和姿态变化的椭圆飞行轨迹。系统在接收到数据后，能够平滑渲染多达 1000 个轨迹点，并在包含近 8 万个三角面的摄影测量场景中保持流畅的帧率（>60 FPS）。

### 1.6 性能提升情况
相比于纯 CPU 驱动的三维计算，基于 OpenGL 硬件加速的渲染管线显著降低了系统的 CPU 占用率，使得界面响应更加迅速，满足了实战级 GCS 对低延迟和高信息密度的要求。

### 1.7 应用场景
该技术可广泛应用于低空物流监控、基础设施巡检、应急救援指挥等需要高精度三维态势感知的领域。

## 2 技术方案正文

### 2.1 项目背景
为了在数字孪生环境中实现无人机的精细化管理，LowAir-GS 提出了基于三维高斯溅射（3D Gaussian Splatting）和传统摄影测量网格的混合渲染架构。Demo-01 作为该架构的最小可行性产品（Minimum Viable Product, MVP），旨在验证数据通信、基础渲染和界面交互的可行性。

### 2.2 研究内容
(1) **UI 交互系统**：设计符合工业级标准的深色主题状态面板，实时显示无人机坐标、欧拉角（Euler Angles）和电池状态。
(2) **通信协议**：定义并实现基于 UDP 的 JSON 遥测接收模块。
(3) **三维渲染引擎**：集成 Assimp 库加载 OBJ 格式的摄影测量模型，实现基础的漫反射光照模型（Diffuse Lighting）。
(4) **数据融合**：将实时坐标映射至三维空间，动态更新无人机位姿矩阵。

### 2.3 研究方法
(1) **模型解析**：利用 Assimp 库的 `aiProcess_Triangulate` 和 `aiProcess_GenSmoothNormals` 标志，将非标准多边形强制转换为三角形，并计算平滑的顶点法向量（Vertex Normals）。
(2) **光照计算**：在 OpenGL 固定管线中，启用 `GL_LIGHT0`。根据兰伯特余弦定律（Lambert's Cosine Law），顶点的漫反射光强 $I_d$ 计算公式为：
$$ I_d = K_d \times \max(0, \mathbf{N} \cdot \mathbf{L}) $$
其中，$K_d$ 为材质的漫反射系数，$\mathbf{N}$ 为顶点法向量，$\mathbf{L}$ 为指向光源的单位向量。
(3) **姿态变换**：无人机的局部坐标系到世界坐标系的变换矩阵 $\mathbf{M}$ 由平移矩阵 $\mathbf{T}$ 和旋转矩阵 $\mathbf{R}$ 组合而成：
$$ \mathbf{M} = \mathbf{T}(x, y, z) \times \mathbf{R}_z(yaw) \times \mathbf{R}_y(pitch) \times \mathbf{R}_x(roll) $$

### 2.4 技术路线
系统的数据流向如下：
(1) 外部设备通过 UDP 端口 14580 广播 JSON 数据报。
(2) `UdpJsonReceiver` 监听到 `readyRead` 信号，读取并反序列化 JSON。
(3) 提取 `uav_id`、坐标、姿态等信息，通过 Qt 信号槽机制（Signal-Slot Mechanism）发送给主线程。
(4) `MainWindow` 更新左侧文本标签，同时调用 `RenderWidget::updateUavState` 更新渲染数据。
(5) `RenderWidget` 触发 `update()`，在下一帧的 `paintGL()` 中，依次调用 `ObjModelRenderer`、`TrajectoryRenderer` 和 `UavRenderer`。

### 2.5 实验设计
实验采用两个具有代表性的真实模型进行验证：
(1) **Stanford Bunny**：真实激光扫描重建模型，包含 35,947 个顶点和 69,451 个三角面，用于验证复杂有机曲面的渲染效果。
(2) **泊松重建地形**：摄影测量风格的地形网格，包含 5,061 个顶点和 9,769 个三角面，用于验证大范围场景的承载能力。
实验中，Python 模拟器以 20Hz 频率发送数据，测试系统在连续运行 1 小时情况下的内存稳定性和渲染帧率。

## 3 项目研究结论及展望

### 3.1 研究结论
实验结果表明，LowAir-GS Demo-01 成功实现了设计目标。系统能够稳定解析 UDP JSON 数据流，基于 Assimp 和 OpenGL 的渲染管线能够高质量地呈现真实的摄影测量三维模型，并将无人机的动态轨迹无缝融合其中。深色主题的紧凑化 UI 设计有效提升了信息密度，符合无人机地面站的实战应用标准。

### 3.2 创新成果
(1) 提出了一种轻量级的虚实融合架构，在不依赖庞大的商业游戏引擎（如 Unreal Engine 或 Unity）的情况下，利用 Qt 原生组件实现了高性能的三维数字孪生底座。
(2) 验证了基于 JSON over UDP 的高频遥测通信方案在本地网络环境下的可靠性。

### 3.3 下一步研究展望
(1) **坐标系转换**：引入 PROJ 库，实现 WGS84 经纬度坐标向局部场景坐标的精确投影转换。
(2) **现代图形管线**：将固定管线升级为基于着色器（Shader-based）的核心模式（Core Profile），以支持 PBR 物理渲染和高分辨率纹理贴图。
(3) **前沿技术集成**：在现有网格渲染的基础上，探索接入三维高斯溅射（3D Gaussian Splatting）渲染器，以实现照片级真实的场景重建与可视化。
