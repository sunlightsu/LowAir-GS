# 任务卡：Demo-01 基础框架与局部坐标系验证

请你接手并推进 GitHub 仓库：https://github.com/SWJTU-AI-Lab/LowAir-GS

当前目标是完成 LowAir-GS 的第一个可运行 Demo。请先阅读仓库根目录 README.md，再重点阅读本文档。

本次任务**不是**做完整 3DGS，**不是**做 COLMAP/ODM 重建，也**不是**做外部摄影测量模型的加载，而是先完成一个具有典型性、展示性、可运行性的最小 Demo：

**Demo-01：UDP 无人机遥测接入与简化低空场景显示**

## 一、任务目标

请实现一个基于 Qt C++ 的桌面端 Demo，能够完成以下功能：
1. 启动 Qt 桌面程序，显示深色主题的 GCS 风格主界面。
2. 显示一个三维场景窗口，场景**仅由程序内置生成**，包含：地面网格、坐标轴、简化建筑方块、起降点、参考航线。
3. 支持接收外部 UDP JSON 无人机实时数据（局部 SCENE 坐标）。
4. 在三维场景中实时显示无人机位置、姿态和历史飞行轨迹。
5. 提供 Python 脚本模拟外部无人机发送实时数据。
6. 形成一个可以演示的完整闭环。

第一版不要追求复杂，核心是把下面这个闭环跑通：

```text
Qt 三维窗口 (内置简化场景)
    ↓
UDP 接收无人机状态 (局部坐标)
    ↓
三维场景中显示无人机位置和轨迹
    ↓
Python 模拟器驱动无人机运动
```

## 二、重要边界与限制（必读）

### 1. 严格的场景来源限制
- **必须使用**程序内部生成的几何体（如 `QOpenGLShaderProgram` 绘制的线和方块）构建场景。
- **严禁引入** Assimp、TinyGLTF 等任何外部模型加载库。
- **严禁加载**外部 OBJ、PLY、GLB 等静态三维资产（这是 Demo-02 的任务）。
- **严禁提交**任何真实摄影测量模型到代码仓库。

### 2. 严格的坐标系限制
- **必须使用**纯局部笛卡尔坐标系（SCENE坐标），即 Python 模拟器发送的 x/y/z 直接对应 OpenGL 场景中的相对位置。
- **严禁进行** WGS84、ENU 到 SCENE 的真实地理坐标系转换（这是 Demo-03 的任务）。可以预留 `CoordinateManager` 接口，但必须直接透传或返回默认值。

### 3. 严格的依赖限制
- **仅允许依赖** Qt6 (Core, Gui, Widgets, OpenGLWidgets, Network)。
- **严禁引入** Assimp、PCL、OSG、GeographicLib、MAVSDK、ROS2 等。

## 三、三维显示与 UI 要求

### 1. 程序内置场景元素
Qt 三维窗口中至少包含以下内置生成的元素：
1. 深蓝灰背景（符合 GCS 标准）
2. 地面网格（如 100x100m，5m 间距）
3. XYZ 坐标轴（红/绿/蓝）
4. 几个简化建筑块（不同高度和颜色的长方体）
5. 起降点或目标点（如地面圆环 + H 标记）
6. 预设参考航线（可选，如黄色虚线椭圆）
7. 无人机当前位置（带姿态旋转的四面体）
8. 历史轨迹线（橙色连线）

### 2. 界面功能与深色主题
必须遵循工业级 GCS UI 设计标准：
- **深色主题**：全局使用 QSS 设置深色背景和浅色文字。
- **状态面板**：显示 UDP 状态、无人机 ID、位置、姿态、电量、轨迹点数。
- **控制按钮**：启动 UDP、停止 UDP、清空轨迹、重置视角。

## 四、UDP JSON 数据格式

Qt 程序监听 UDP 端口 `14580`，接收 SCENE 局部坐标：

```json
{
  "uav_id": "sim_01",
  "timestamp_ms": 1710000000123,
  "frame": "SCENE",
  "x": 30.0,
  "y": 15.0,
  "z": -20.0,
  "roll": 0.0,
  "pitch": 0.0,
  "yaw": 90.0,
  "battery": 88.0
}
```

## 五、Python 无人机模拟器

实现 `tools/telemetry_simulator/send_uav_udp.py`：
- 模拟无人机沿椭圆轨迹飞行。
- 支持命令行参数 `--rate` (发送频率)、`--host`、`--port`。

## 六、验收标准

1. **零外部三维库依赖**：CMakeLists.txt 中除了 Qt6 无其他依赖，不链接 Assimp。
2. **场景内置**：场景完全由 `SimpleScene.cpp` 硬编码绘制，无需读取任何外部文件。
3. **坐标透传**：`CoordinateManager` 明确注释 Demo-01 仅使用局部坐标。
4. **演示闭环**：启动程序 -> 点击 Start UDP -> 运行 Python 模拟器 -> 场景中出现运动的无人机和轨迹线。
5. **文档完备**：包含 `install.sh`、用户手册、二次开发手册、技术报告，且所有文档均明确 Demo-01 的功能边界。
