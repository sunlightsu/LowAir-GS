# LowAir-GS 演示路线图 (Demo Roadmap)

为了保证项目的工程化落地质量，LowAir-GS 采取“小步快跑、边界清晰”的迭代策略。本项目规划了 7 个核心 Demo，每个 Demo 都有严格的边界限制和明确的验收标准。

---

## Demo 独立性原则

LowAir-GS 后续每一个 Demo-N 都必须保持独立演示能力。不能把所有功能都堆叠到 Demo-01 中，也不能让后续 Demo 只能依赖前一个 Demo 的隐式状态运行。

每个 Demo 必须具备：

1. **独立目录**；
2. **独立 README**；
3. **独立安装说明**；
4. **独立运行命令**；
5. **独立输入数据说明**；
6. **独立功能边界**；
7. **独立验收标准**；
8. **与前序 Demo 的关系说明**。

如果某个 Demo 复用了前序 Demo 的代码，应在 README 中明确说明复用模块和新增模块；如果需要前置依赖，也必须提供清晰的安装与运行步骤。

---

## 目录结构规划

遵循上述独立性原则，后续 Demo 的目录结构规划如下：

```text
apps/
├── demo01_udp_uav_scene/           # Demo-01 (当前暂存于 uav_scene_fusion_qt)
├── demo02_static_asset_viewer/     # Demo-02 (实际目录名)
├── demo03_geo_coordinate_alignment/# Demo-03
├── demo04_real_uav_bridge/         # Demo-04
├── demo05_gaussian_splat_viewer/   # Demo-05
├── demo06_dual_scene_fusion/       # Demo-06
└── demo07_game_event_evaluation/   # Demo-07
```

*注：当前 Demo-01 暂存于 `apps/uav_scene_fusion_qt/`，后续 Demo-02 起建议严格拆分为独立目录，避免功能边界混乱。*

## Demo-01: UDP 无人机遥测接入与简化低空场景显示 (已完成)

**核心目标：** 跑通 Qt + OpenGL 渲染管线，验证 UDP 遥测接收与 UI 状态更新机制。

**功能边界：**
- [x] **场景来源**：程序内部硬编码的简化低空场景（网格、坐标轴、建筑块、起降点、参考航线）。
- [x] **坐标系统**：使用纯局部笛卡尔坐标系（SCENE坐标），不做任何真实地理坐标转换。
- [x] **数据来源**：Python 模拟器通过 UDP JSON 发送模拟遥测数据。
- [x] **外部依赖**：**零外部三维库依赖**（仅依赖 Qt6），无需 Assimp、PCL、OSG 等。

---

## Demo-02: 静态三维资产加载与渲染 (已完成)

**核心目标：** 引入第三方模型加载库，实现外部静态三维资产（摄影测量网格）的加载与渲染。

**功能边界：**
- [x] **场景来源**：外部 OBJ / PLY / GLB 格式的摄影测量或手工建模资产。
- [x] **外部依赖**：引入 Assimp 等模型解析库。
- [x] **渲染能力**：实现基础的光照、材质解析和线框模式渲染。
- [x] **限制**：仍使用局部坐标系，模型位置通过手动偏移（Offset）对齐。

---

## Demo-03: WGS84 / ENU / SCENE 坐标转换与真实位置对齐 (已完成)

**核心目标：** 解决真实世界坐标与 OpenGL 局部渲染坐标的对齐问题。

**功能边界：**
- [x] **坐标系统**：实现 WGS84（经纬高） → ENU（东北天） → SCENE（OpenGL局部）的完整转换链路。
- [x] **外部依赖**：引入内置 WGS84 椭球转换公式（无需重型外部库，适合局部场景）。
- [x] **数据来源**：Python 模拟器发送带有真实经纬度的遥测数据。
- [x] **限制**：不接入真实飞行器，仅处理坐标数学转换。

---

## Demo-04: MAVLink 无人机遥测接入与三维显示 (已完成)

**核心目标：** 替换自定义的 UDP JSON 协议，接入工业界标准无人机通信协议。

**功能边界：**
- [x] **协议支持**：实现 MAVLink 协议解析（如接入 PX4/ArduPilot 飞控）。
- [x] **数据来源**：SITL（软件在环仿真）或真实无人机硬件。
- [x] **外部依赖**：引入 MAVLink Raw 解析（兜底实现）。
- [x] **限制**：仅做遥测数据下行（Telemetry），不做上行控制（Control）。

---

## Demo-05: 3D Gaussian Splatting 场景加载与显示 (已完成)

**核心目标：** 实现标准 3D Gaussian Splatting 显式高斯场景资产的加载与实时展示（注：3DGS 是显式高斯表示，不是隐式神经辐射场 NeRF）。

**功能边界：**
- [x] **场景来源**：由 COLMAP / 预训练模型生成的 `.splat` 或 `.ply` 格式的 3DGS 资产。
- [x] **渲染能力**：实现基于 CPU 排序或深度测试的基础 OpenGL Billboard Splat 渲染，支持 Point/Splat 双模式切换。
- [x] **性能指标**：提供实时 FPS、帧耗时和 VRAM 显存占用估算统计面板。
- [x] **限制**：为保持轻量和跨平台，当前未使用 CUDA 官方光栅化器。与摄影测量网格的双源融合（Mesh+GS）保留至 Demo-06 实现。

---

## Demo-06: 摄影测量模型 + 3DGS 双源场景融合显示 (已完成)

**核心目标：** 在同一三维场景中同时展示传统摄影测量网格模型和 3DGS 显式高斯场景，实现双源异构渲染融合。

**功能边界：**
- [x] **双源渲染**：将 Demo-02 的多边形网格渲染与 Demo-05 的 3DGS 体积渲染在同一帧内层叠展示。
- [x] **独立坐标对齐**：支持对 Mesh 和 Gaussian 资产进行独立的平移、旋转、缩放对齐控制。
- [x] **四种融合模式**：支持在 Mesh Only、Gaussian Only、Dual Fusion、Wireframe Comparison 四种模式之间实时切换。
- [x] **性能基准**：建立双源融合场景下的实时性能统计面板（顶点数、面片数、高斯数量、FPS、VRAM估算）。

---

## Demo-07: 虚拟目标、事件、特效与游戏化评估 (已完成)

**核心目标：** 在完整虚实融合场景中引入任务式交互元素，实现教学展示和游戏化评估功能。

**功能边界：**
- [x] **虚拟目标**：在三维场景中放置可交互的虚拟目标点（如车辆、建筑、兴趣点）。
- [x] **事件触发**：无人机进入指定区域时触发可视化事件（如闪烁、标注弹出、脉冲动画）。
- [x] **特效与增强渲染**：实现粒子爆发、Glow发光、Label Popup弹窗、Path Highlight轨迹高亮以及Zone Outline区域脉冲动画。
- [x] **游戏化评估**：对无人机完成任务的效率、路径、电量消耗进行自动评分和可视化展示。
