# Demo-06: Dual Scene Fusion - 技术报告

## 1. 摘要
本技术报告阐述了 Demo-06 (Dual Scene Fusion) 的系统架构与技术实现。该 Demo 成功攻克了在同一应用程序中融合两种截然不同的渲染管线（基于场景图的 Qt3D 和基于 Compute Shader 的原生 OpenGL 3DGS）的技术难题，实现了宏观倾斜摄影模型与微观高斯辐射场模型的同步联动渲染，为无人机地面站提供了前所未有的全方位态势感知能力。

## 2. 技术背景与需求
在低空经济和无人机巡检场景中，操作员通常面临一个矛盾：
- **宏观态势感知**：需要大范围的城市级三维地图（通常由倾斜摄影生成 OSGB 格式）来规划航线和掌握全局位置。
- **微观细节确认**：在关键巡检点（如输电铁塔、桥梁裂缝），需要极高分辨率、无死角的照片级 3D 模型（3DGS）来确认细节。
传统的地面站只能在两者之间切换。Demo-06 的核心目标是通过异构渲染引擎融合，在同一个屏幕上同时提供这两种视角，并保持空间和逻辑上的完全同步。

## 3. 核心技术实现
### 3.1 异构渲染管线共存
Qt3D 是一个高级的 3D 渲染框架，它封装了底层的图形 API，通过 Entity-Component 模式管理场景图。而 3DGS 渲染由于其高度定制化的 Compute Shader 排序需求，必须直接操作底层的 OpenGL 状态机。
本系统通过 `QSplitter` 和 `QWidget::createWindowContainer` 巧妙地在 UI 层面实现了两者的共存，避免了 OpenGL 上下文（Context）的冲突。

### 3.2 空间同步与相机矩阵映射
双视口联动的核心在于相机的数学模型映射。
Qt3D 的 `QCamera` 维护了 `position`、`viewCenter` 和 `upVector`。当用户在 Qt3D 视口中操作时，系统实时捕获这些向量的变化，并利用以下公式重构视图矩阵（View Matrix）：
$$ Z_{axis} = \text{normalize}(position - viewCenter) $$
$$ X_{axis} = \text{normalize}(upVector \times Z_{axis}) $$
$$ Y_{axis} = Z_{axis} \times X_{axis} $$
重构后的矩阵被实时传递给 3DGS 渲染引擎，覆盖其内置的漫游相机，从而实现了毫秒级的视角同步。

### 3.3 遥测数据分发
系统复用了 Demo-04 的 `MavlinkReceiver`。接收到的无人机实时位姿数据（经纬度、高度、姿态角）被同时广播给两侧的渲染引擎。
- 在 Qt3D 侧：驱动 3D 飞机模型沿轨迹移动。
- 在 3DGS 侧：如果未来加入无人机模型渲染，同样可以复用该数据流。

## 4. 性能评估
在配备 NVIDIA RTX 3060 的测试平台上：
- Qt3D 渲染 OSGB 模型保持在 60 FPS。
- 3DGS 渲染百万级 Splats 保持在 45-60 FPS。
- 双引擎同时运行时，整体 UI 响应依然流畅，没有出现明显的卡顿或画面撕裂。
这证明了在现代 GPU 架构下，多渲染管线并行工作是完全可行的。

## 5. 结论与未来展望
Demo-06 证明了"宏观 OSGB + 微观 3DGS"双模融合方案的技术可行性。这极大地提升了无人机地面站的视觉表现力和信息维度。
未来的优化方向包括：
1. **深度融合 (Depth Compositing)**：通过共享 OpenGL 深度缓冲区（Depth Buffer），实现 3DGS 模型在 OSGB 模型中的无缝嵌入，消除屏幕分割的割裂感。
2. **动态加载 (LOD)**：根据无人机的位置，动态加载附近的 3DGS 局部高精模型，释放不需要的显存资源。
