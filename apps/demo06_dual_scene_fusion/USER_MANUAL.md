# Demo-06: Dual Scene Fusion - 用户手册

## 1. 简介
本 Demo 是整个系统的一个重要里程碑。它成功将传统的倾斜摄影模型（OSGB，基于 Qt3D 渲染）与最新的 3D Gaussian Splatting 模型（3DGS，基于原生 OpenGL 渲染）无缝融合在一个界面中，并接入了无人机真实 MAVLink 遥测数据。用户可以在双视口中同步观察无人机在宏观城市环境和微观高保真环境中的飞行状态。

## 2. 系统要求
- 操作系统：Ubuntu 20.04/22.04/24.04
- 依赖环境：Qt 5.15+ (包含 Qt3D)、OpenGL 4.3+
- 硬件要求：**必须配备独立显卡**（支持 Compute Shader），推荐显存 4GB 以上。

## 3. 安装与运行
### 3.1 一键安装
在项目根目录或本目录下执行安装脚本：
```bash
./install.sh
```

### 3.2 运行程序
安装完成后，在 `build` 目录下运行可执行文件：
```bash
cd build
./Demo06DualSceneFusion
```

## 4. 操作说明
### 4.1 界面布局
程序界面分为左右两个主要的 3D 视图：
- **左侧视口 (Qt3D View)**：用于渲染宏观的 OSGB 倾斜摄影模型和无人机轨迹。
- **右侧视口 (3DGS View)**：用于渲染微观局部的 3D Gaussian Splatting 高保真模型。
- **控制面板 (底部/浮动)**：包含 MAVLink 连接控制、相机同步开关、模型加载等。

### 4.2 数据接入
1. 点击控制面板上的 **Start UDP (14550)** 按钮，开始监听 MAVLink 数据。
2. 运行遥测数据模拟器发送轨迹数据：
   ```bash
   python3 ../../../tools/telemetry_simulator/send_mavlink_udp.py
   ```
3. 接收到数据后，无人机模型将在左侧的 OSGB 场景中开始移动。

### 4.3 视图同步与漫游
- **独立漫游**：默认情况下，左右视口的相机是独立的。您可以在左侧或右侧视口中按住鼠标左键拖动以旋转视角，滚动滚轮缩放。
- **相机同步 (Sync Camera)**：
  - 勾选控制面板上的 **Sync Camera** 选项。
  - 此时，右侧 3DGS 视口的相机将完全锁定并跟随左侧 Qt3D 视口的相机。
  - 这允许您在宏观场景中导航时，右侧窗口自动呈现对应位置的高保真细节。

## 5. 常见问题
- **Q: 为什么右侧的 3DGS 场景是黑屏或无法显示？**
  - A: 请确保您的显卡驱动支持 OpenGL 4.3。另外，由于两个渲染引擎共存，显存占用较大。如果显存不足，可能导致 3DGS 模型加载失败。
- **Q: 相机同步时，右侧视角看起来不对？**
  - A: 确保左侧 OSGB 模型和右侧 3DGS 模型处于同一个世界坐标系下（参考 Demo-03 的坐标对齐流程）。如果它们在物理空间上不重合，同步相机时右侧可能只会看到空白。
