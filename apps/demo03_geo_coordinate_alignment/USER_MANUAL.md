# Demo-03: Geo-Coordinate Alignment - 用户手册

## 1. 简介
本 Demo 演示了地理坐标系（WGS84 经纬度）与局部笛卡尔坐标系（ENU）之间的转换与对齐。通过输入基准点和目标点的经纬度，程序能够计算出目标点在局部坐标系中的位置，并在 3D 场景中进行可视化展示。

## 2. 系统要求
- 操作系统：Ubuntu 20.04/22.04/24.04
- 依赖环境：Qt 5.15+ (包含 Qt3D)、GeographicLib 2.3+

## 3. 安装与运行
### 3.1 一键安装
在项目根目录或本目录下执行安装脚本（会自动安装依赖的 GeographicLib）：
```bash
./install.sh
```

### 3.2 运行程序
安装完成后，在 `build` 目录下运行可执行文件：
```bash
cd build
./Demo03GeoCoordinateAlignment
```

## 4. 操作说明
### 4.1 界面布局
- **左侧控制面板**：包含基准点（Origin）设置、目标点（Target）设置、计算结果显示和坐标系说明。
- **右侧 3D 视图**：可视化展示基准点、目标点、坐标轴（ENU）和连接线。

### 4.2 坐标转换操作
1. **设置基准点**：在左侧面板的 "Origin (WGS84)" 区域，输入基准点的纬度（Lat）、经度（Lon）和海拔（Alt）。默认值为西南交大某位置。
2. **设置目标点**：在 "Target (WGS84)" 区域，输入目标点的纬度、经度和海拔。
3. **计算对齐**：点击 **Align Coordinates** 按钮。
4. **查看结果**：
   - "Result (ENU Local)" 区域将显示目标点相对于基准点的局部坐标 (X, Y, Z)，单位为米。
   - 3D 视图中将更新目标点（红色球体）的位置，并绘制从基准点到目标点的虚线。

### 4.3 3D 视图交互
- **旋转视角**：按住鼠标左键并拖动。
- **缩放视角**：滚动鼠标滚轮。
- **平移视角**：按住鼠标右键并拖动。
- 坐标轴说明：红色代表 East (X)，绿色代表 North (Y)，蓝色代表 Up (Z)。

## 5. 常见问题
- **Q: 编译时提示找不到 GeographicLib？**
  - A: 请确保执行了 `sudo apt-get install libgeographic-dev`，或运行了本目录下的 `install.sh`。
- **Q: 计算出的距离与实际不符？**
  - A: 本程序使用严密的椭球体模型（WGS84）进行计算，结果比简单的球面模型更精确。如果误差较大，请检查输入的经纬度格式是否为十进制度数（Decimal Degrees）。
