# Demo-02: Static Asset Viewer - 技术报告

## 1. 摘要
本技术报告详细说明了 Demo-02 (Static Asset Viewer) 的技术实现细节。该 Demo 验证了在 Qt3D 框架下加载、解析和渲染外部静态 3D 资产（如无人机模型、场景建筑）的能力，是构建复杂 3D 场景的基础组件。

## 2. 系统架构
系统主要由表现层和渲染层构成：
- **表现层**：`MainWindow` 提供用户交互界面，允许用户选择本地文件系统中的 3D 模型文件。
- **渲染层**：`RenderWidget` 利用 Qt3D 的 `QSceneLoader` 组件异步加载模型文件，并将其解析为 Qt3D 的实体-组件系统（ECS）结构进行渲染。

## 3. 关键技术实现
### 3.1 模型异步加载
为了避免在加载大型 3D 模型时阻塞主线程导致界面卡顿，系统采用了 Qt3D 的 `QSceneLoader`。该组件在后台线程解析模型文件（如 `.obj`, `.gltf`），解析完成后自动将生成的几何体（Geometry）、材质（Material）和变换（Transform）组件附加到指定的根实体上。

### 3.2 场景图自适应
由于外部模型的尺度和坐标系可能各不相同，系统在模型加载完成后，需要对相机的视场和模型的位置进行自适应调整。虽然当前的实现较为基础（固定相机参数），但在实际的资产查看器中，通常会通过计算模型包围盒（Bounding Box）来动态调整相机位置，确保模型始终居中且完整显示。

### 3.3 材质与光照映射
Qt3D 默认支持 Phong 光照模型。当加载带有材质定义（如 `.mtl` 文件）的模型时，`QSceneLoader` 会自动将其映射为 `QPhongMaterial` 或相应的材质组件。为了保证模型在各个角度都能被清晰观察，场景中配置了一个随相机移动的定向光（Directional Light）。

## 4. 性能分析
- **加载速度**：对于中等复杂度（10万面以内）的模型，加载时间在百毫秒级；对于复杂模型，异步加载机制确保了 UI 的响应性。
- **渲染性能**：得益于 Qt3D 的底层优化（如视锥体裁剪、状态排序），静态模型的渲染帧率非常稳定。

## 5. 结论与展望
Demo-02 成功实现了外部静态 3D 资产的加载与展示。后续的 Demo 将结合 Demo-01 的动态遥测数据，实现动态模型与静态场景的融合，并逐步引入更高阶的渲染技术（如基于高斯溅射的场景重建）。
