# Demo-07: Game Event Evaluation - 技术报告

## 1. 摘要
本技术报告详细阐述了 Demo-07 (Game Event Evaluation) 的技术架构与实现。该 Demo 在 3D Gaussian Splatting (3DGS) 高保真渲染引擎的基础上，成功集成了一套轻量级的 3D 游戏事件引擎与视觉特效系统。通过自动化的空间碰撞检测与多层次的视觉反馈，系统能够对无人机的巡检轨迹进行实时的自动化评估，极大地提升了地面站软件的交互性与实用性。

## 2. 核心系统架构
Demo-07 的架构可以分为三个核心子系统：
1. **渲染子系统 (Rendering Subsystem)**：以 `GaussianSplatWidget` 为核心，负责底层的 3DGS 场景渲染。
2. **事件引擎 (Event Engine)**：以 `GameEventEngine` 为核心，负责空间计算、碰撞检测和状态机管理。
3. **特效管线 (VFX Pipeline)**：以 `Effect` 基类及其派生类为核心，负责生命周期管理和 OpenGL 特效绘制。

## 3. 关键技术实现
### 3.1 空间碰撞检测与事件触发
无人机的轨迹回放由 `TrajectoryReplay` 模块以固定频率（通常 60Hz）提供实时的 3D 坐标。`GameEventEngine` 在每一帧执行空间碰撞检测：
$$ D = \sqrt{(x_{uav} - x_{target})^2 + (y_{uav} - y_{target})^2 + (z_{uav} - z_{target})^2} $$
当距离 $D$ 小于目标设定的触发半径 $R$ 时，且该目标未被触发过，引擎将产生一个 `TargetTriggered` 事件。这种基于欧氏距离的检测算法在处理数百个目标时依然能保持极高的效率。

### 3.2 复合视觉特效 (Virtual Explosion)
为了提供强烈的视觉反馈，系统实现了一套基于原生 OpenGL 的复合特效管线。以最复杂的 `VirtualExplosionRenderer` 为例，它在单次 Draw Call 或少量的 Draw Calls 中融合了五种视觉层次：
1. **中心闪光**：通过 Point Sprite 结合高斯衰减 Alpha 纹理实现。
2. **环形波纹**：利用 Fragment Shader 中的正弦函数 $\sin(d - t)$ 计算距离中心的波动相位，生成多层同心圆。
3. **动态粒子**：通过 CPU 计算抛物线运动方程，利用 VBO 动态更新顶点数据，模拟粒子喷发。
4. **烟雾扩散**：使用大尺寸的半透明 Point Sprite，结合缓慢的向外扩散速度向量。
5. **地面发光环**：在 Y=0 的平面上绘制带有脉冲动画的线框。
所有这些特效都拥有独立的生命周期管理，并随着时间的推移自动计算透明度衰减，最终由渲染管线自动回收。

### 3.3 2D HUD 与 3D 场景的混合渲染
系统需要在 3D 场景（深度测试开启）之上绘制 2D 的文字弹窗（Label Popup）和 UI 元素。
技术实现上，首先完成所有的 3D 渲染（包括 3DGS 和 3D 特效），然后禁用深度测试 (`glDisable(GL_DEPTH_TEST)`)，最后利用 `QPainter` 绑定到当前的 `QOpenGLPaintDevice` 上进行 2D 绘制。这确保了 2D HUD 永远位于屏幕的最前端，且不受 3D 场景光照和深度的影响。

## 4. 性能分析
引入事件引擎和特效管线后，对系统性能的额外开销进行了评估：
- **碰撞检测**：由于目标数量有限（通常 < 1000），每帧的距离计算耗时 < 0.1ms。
- **特效渲染**：即使在触发最复杂的虚拟爆炸特效时，由于采用了高效的 Point Sprite 和极简的 Shader 逻辑，帧率下降幅度 < 5%，整体依然保持在流畅的水平。

## 5. 结论
Demo-07 成功地将游戏开发中的事件驱动机制和视觉特效技术引入到了专业的无人机地面站软件中。这不仅使得无人机巡检任务的评估过程变得直观和自动化，也为未来开发更复杂的数字孪生应用（如模拟对抗、应急演练）奠定了坚实的技术基础。
