# LowAir-GS Demo-01 软件应用手册

## 1 软件概述

LowAir-GS Demo-01 是一款基于 Qt6 和 OpenGL 开发的摄影测量三维模型与外部无人机实时数据虚实融合显示系统。该系统主要用于在三维数字孪生环境中，实时接收并可视化无人机的遥测数据，为无人机地面站（Ground Control Station, GCS）提供直观的态势感知能力。

本手册旨在指导终端用户完成软件的安装、启动和日常操作。

## 2 运行环境与安装

### 2.1 软硬件要求

| 项目 | 要求 |
|------|------|
| 操作系统 | Ubuntu 22.04 LTS 或 24.04 LTS |
| 编译器 | GCC/G++ 11 或更高版本 |
| 构建工具 | CMake 3.16 或更高版本 |
| 核心依赖 | Qt 6 (Core, Gui, Widgets, OpenGLWidgets, Network) |
| 三维依赖 | OpenGL (libgl1-mesa-dev), Assimp 5.3.0 |
| 脚本环境 | Python 3.10+ (用于运行模拟器) |

### 2.2 一键安装说明

为了简化环境配置，项目根目录提供了一键安装脚本 `install.sh`。该脚本会自动执行以下操作：
(1) 更新系统软件包列表。
(2) 安装 Qt6、CMake 和编译工具链。
(3) 从源码下载并编译安装 Assimp 5.3.0 库。
(4) 编译 LowAir-GS Demo-01 Qt 主程序。

**执行安装：**
在终端中进入项目根目录，运行以下命令：
```bash
chmod +x install.sh
./install.sh
```

## 3 软件操作指南

### 3.1 启动系统

安装完成后，主程序可执行文件将生成在 `apps/uav_scene_fusion_qt/build/UavSceneFusionQt` 路径下。

在终端中运行：
```bash
./apps/uav_scene_fusion_qt/build/UavSceneFusionQt
```

### 3.2 主界面布局

软件界面采用符合工业级地面站标准的深色主题（Dark Mode），主要分为左右两个区域：
(1) **左侧控制面板**：包含系统控制按钮、网络状态指示、实时遥测数据面板和操作提示。
(2) **右侧三维视图**：基于 OpenGL 渲染的数字孪生场景，包含真实的摄影测量模型（如斯坦福兔子扫描模型和泊松重建地形）、地面网格、坐标轴以及无人机实体。

### 3.3 建立遥测连接

(1) 在左侧控制面板的 **Control** 分组中，点击 `Start UDP (14580)` 按钮。
(2) 观察 **UDP Status** 分组，状态应由灰色的 `Stopped` 变为绿色的 `Listening on :14580`。
(3) 此时，系统已准备好接收来自外部无人机或模拟器的遥测数据。

### 3.4 启动无人机模拟器（测试用）

如果当前没有真实的无人机硬件连接，可以使用内置的 Python 模拟器生成测试数据。
打开一个新的终端窗口，在项目根目录下运行：
```bash
python3 tools/telemetry_simulator/send_uav_udp.py
```
模拟器默认将以 20 Hz 的频率，向本地 `14580` 端口发送带有椭圆飞行轨迹的无人机状态数据。

### 3.5 态势监控与视图控制

当系统成功接收到数据后：
(1) **状态更新**：UDP 状态将显示为绿色的 `Receiving Data`。
(2) **遥测面板**：实时更新无人机 ID、三维坐标（Position）、姿态角（Attitude）和电池电量（Battery）。当电量低于 20% 时，电量数值将变为黄色高亮警告。
(3) **三维视图**：右侧场景中将出现绿色的四面体无人机模型，并随数据实时移动和旋转。同时，无人机尾部会生成橙色的历史飞行轨迹线（最多保留 1000 个点）。

**鼠标视图控制：**
(1) **旋转视角**：在三维视图中按住鼠标左键并拖动。
(2) **缩放视角**：滚动鼠标滚轮。
(3) **重置视角**：点击左侧面板的 `Reset View` 按钮，相机将恢复到默认的全局俯视角度。

### 3.6 停止与清理

(1) **清理轨迹**：点击 `Clear Trajectory` 按钮，可清空当前三维场景中的橙色轨迹线，并将轨迹点计数归零。
(2) **停止接收**：点击 `Stop UDP` 按钮，系统将关闭 14580 端口，停止接收新数据。

## 4 常见问题解答 (FAQ)

**Q: 启动程序时提示找不到 libassimp.so.5？**
A: 请确保 Assimp 已正确安装，并且系统库缓存已更新。可以尝试在终端中运行 `sudo ldconfig`，或检查 `LD_LIBRARY_PATH` 环境变量。

**Q: 三维场景中看不到无人机？**
A: 请确认是否已点击 `Start UDP`，并且模拟器正在运行。如果面板上有数据但场景中不可见，请尝试滚动鼠标滚轮缩小视图，或点击 `Reset View`。

**Q: 如何接入真实的无人机数据？**
A: 真实无人机的机载计算机或伴飞计算机需要将飞控状态封装为系统规定的 JSON 格式，并通过 UDP 发送到地面站运行主机的 14580 端口。具体协议格式请参考《二次开发手册》。
