# Demo-07 示例数据 (Demo-07 Sample Data)

本目录包含了用于 Demo-07 (虚拟目标、事件、特效与游戏化评估) 的示例数据文件。这些文件由 Python 脚本自动生成，用于演示任务配置、虚拟目标、触发区域、轨迹回放以及评分规则。

## 文件说明

1. **`mission_config.json`**
   - 任务主配置文件，定义了任务的基本信息（名称、描述）、关联的各个子配置文件的路径，以及回放控制参数。
   - 程序启动时，只需加载此文件，程序会自动解析并加载其他关联文件。

2. **`virtual_targets.json`**
   - 虚拟目标配置文件，定义了 3D 场景中的多个目标点。
   - 每个目标包含：ID、名称、3D 坐标位置 (x, y, z)、半径大小以及关联的视觉特效类型（支持 `glow` 发光特效和 `particle` 粒子爆发特效）。

3. **`trigger_zones.json`**
   - 触发区域配置文件，定义了 3D 场景中的特定区域（如禁飞区或重点巡检区）。
   - 每个区域包含：ID、名称、类型、中心点坐标、尺寸 (宽度、高度、深度) 以及关联的视觉特效类型（如 `pulse`）。

4. **`replay_track.jsonl`**
   - 无人机轨迹回放数据文件，采用 JSONL (JSON Lines) 格式，每行代表一帧状态。
   - 包含数据：时间戳 (`t`)、3D 位置 (`pos`)、姿态角 (`yaw`, `pitch`, `roll`) 以及模拟电量 (`battery`)。
   - 帧率为 10Hz（每秒10帧），总时长约 60 秒。

5. **`score_rules.json`**
   - 评分规则配置文件，定义了游戏化评估的加减分逻辑。
   - 包含基础分数、各类型事件的得分权重（如到达目标加分、进入禁飞区扣分），以及时间、电量等消耗性指标的惩罚系数。

6. **`event_records.json`**
   - 事件记录文件（由程序运行后生成或导出），记录了任务执行过程中触发的所有事件。
   - 用于任务结束后的复盘和详细分析。

7. **`mission_report.json`**
   - 任务最终报告文件（由程序运行后生成或导出），包含了任务的最终得分、各项评分明细以及整体评估结果。

## 数据生成方式

本目录下的初始数据文件（配置文件和轨迹数据）由仓库工具目录中的 Python 脚本生成。如需重新生成或修改生成逻辑，请运行：

```bash
python3 tools/gaussian_generators/generate_demo07_assets.py
```

本目录下的所有数据均为纯数学模拟生成的虚拟测试数据，不对应任何真实地理坐标、真实飞行器轨迹或真实任务场景。仅供软件功能演示与测试使用，不涉及任何真实无人机控制。
